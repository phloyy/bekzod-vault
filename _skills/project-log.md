# SKILL: project-log

## Назначение
Создание и ведение лога задач по проекту внутри vault.

---

## Входные данные
- `project` — название проекта
- `date` — дата записи
- `tasks` — список задач / что сделано
- `blockers` — что мешает (опционально)
- `next` — следующие шаги

---

## Структура выходного файла

```md
---
created: {{date}}
tags: [log, {{project}}]
status: active
project: {{project}}
---

# {{project}} — Лог

## {{date}}

### Сделано
- [ ] 
- [ ] 

### Блокеры
- 

### Следующий шаг
- 

---
```

## Правила

- Один файл = один проект, записи добавляются в конец (append)
- Если файл уже существует — добавь новый блок `## {{date}}` в конец
- Используй `[[]]` ссылки на смежные файлы проекта
- Статус меняй: `draft → active → completed → archived`

---

## Именование файла

```
projects/{{project}}/log.md
```

Пример: `projects/NOFILLER/log.md`
